🔍 HTML显示问题诊断工具

📋 问题描述

某些HTML文件在R&D Report模块中只能显示背景，无法显示内容。这通常是由以下原因造成的：

  • ✅ CSS样式冲突 - 原HTML的样式被覆盖
  • ✅ 元素隐藏 - display:none 或 visibility:hidden
  • ✅ JavaScript问题 - 脚本执行失败
  • ✅ 资源路径问题 - 相对路径的CSS/JS无法加载
  • ✅ iframe限制 - 沙箱安全策略

🛠️ 诊断步骤

1 打开浏览器开发者工具

按 F12 或右键选择"检查元素"打开开发者工具

2 查看Console标签页

查看是否有以下类型的错误信息：

// CSS相关错误 Failed to load resource: net::ERR_FILE_NOT_FOUND (style.css) // JavaScript错误 Uncaught ReferenceError: $ is not defined // 控制台日志 🔍 HTML Analysis: {hasHead: true, hasBody: true, ...} 📄 Iframe loaded, setting up content...
3 检查Elements标签页

在iframe中查找body元素，检查是否有以下问题：

  • display: none - 元素被隐藏
  • visibility: hidden - 元素不可见
  • opacity: 0 - 元素透明
  • position: fixed - 定位问题
4 测试原始HTML文件

直接在浏览器中打开原始HTML文件，确认是否正常显示

✅ 已实施的修复

  • 增强的CSS重置 - 强制显示所有隐藏元素
  • iframe调试信息 - 右上角显示HTML字符数
  • 内容分析工具 - 自动分析HTML结构
  • 样式覆盖 - 使用!important强制可见性
  • 路径处理 - 警告相对路径问题

⚠️ 常见问题及解决方案

问题1: 背景显示但内容不显示

// 可能原因：CSS样式隐藏了内容 body { display: none; } .content { visibility: hidden; } // 解决方案：系统已自动修复 [style*="display: none"] { display: block !important; }

问题2: 外部CSS/JS无法加载

// 可能原因：相对路径在iframe中无法解析 <link href="css/style.css" rel="stylesheet"> // 解决方案：检查控制台警告并修复路径 ⚠️ Found relative CSS path, may not load correctly: css/style.css

问题3: JavaScript错误

// 可能原因：jQuery等库未加载 Uncaught ReferenceError: $ is not defined // 解决方案：检查并修复脚本依赖
